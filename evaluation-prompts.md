# BondView 评测 Prompt 集

> 面向 BondView 债券行情聚合系统的任务描述，共 47 题。Feature 迭代与 0-1 模块题均可在前端页面完成验收；测试、重构、工程化题为一句话指令。

---

## Feature 迭代（20 题）

**F-01 [Feature迭代]**  
聚合行情列表页目前只展示债券静态信息，交易员希望在 `/market` 表格中直接看到各源汇总后的最优买价、最优卖价、买卖价差（BP）以及有效报价源数量。列表需支持按价差升序/降序排序，价差列用颜色区分宽窄（例如小于 3BP 为绿色、大于 10BP 为红色）。点击某行价差应跳转到该债券详情页并自动定位到「各源报价对比」Tab。请基于现有 `/api/bonds` 与 `/api/quotes/best` 或 `/api/bonds/{id}/aggregated` 接口完成前后端改造，并修复当前表格中 `type`、`rating` 字段与后端 `bond_type`、`credit_rating` 不一致导致列空白的问题。

**F-02 [Feature迭代]**  
债券详情页 `BondDetail.vue` 缺少设计文档中的收益互换信息。请在详情页新增「收益互换」Tab：展示与该债券关联的全部互换报价，列包括交易商、方向、互换利率、期限、最小名义本金、报价时间；支持按交易商名称搜索、按期限下拉筛选；对同一期限下最优报价行高亮。若该债券暂无互换数据，Tab 内展示友好空态并提供返回聚合行情的链接。期限与金额字段需使用后端 `tenor`、`notional_min`，不要沿用前端旧字段名。

**F-03 [Feature迭代]**  
国债期货与现券的基差关系是交易员关注重点。请在债券详情页新增「关联期货」Tab：根据债券类型与剩余期限匹配可交割的国债期货合约（如 T/TF/TS），展示合约代码、最新价、涨跌幅、基差、对应期货源名称；基差为负与为正时使用不同颜色。Tab 顶部增加简短说明文案解释基差含义。点击合约代码应跳转到 `/futures` 页并自动筛选到对应合约类型。数据可来自现有 `/api/futures/contracts` 与 `/api/futures/basis` 接口并按债券维度做关联。

**F-04 [Feature迭代]**  
成交记录页 `/trades` 已有统计卡片和表格，但后端 `/api/trades/statistics` 返回的按行情源、按债券类型分布数据尚未可视化。请在该页上半区增加 ECharts 图表区：左侧饼图展示各行情源成交占比，右侧柱状图展示各债券类型成交量；图表需与下方表格筛选联动——点击饼图扇区或柱状图条目时，表格自动按对应维度过滤，并提供「清除图表筛选」按钮。图表在窄屏下应纵向堆叠，保持金融终端风格配色。

**F-05 [Feature迭代]**  
看板 `/dashboard` 的国债收益率曲线目前为静态展示。请增强交互：鼠标悬停期限点显示该期限收益率、较上一日变动 BP；增加「国债 / 政金债」曲线切换或叠加对比；增加「近 7 日 / 近 30 日」时间范围切换（近 30 日可用模拟历史点）。曲线区域增加 loading 与空数据状态。所有交互结果应即时反映在图表上，无需整页刷新。

**F-06 [Feature迭代]**  
「我的关注」页 `/favorites` 功能较单薄。请在保留取消关注能力的基础上，为每只收藏债券展示最新最优买价、最优卖价、较收藏时价格变动（涨跌 BP 及箭头）、最后更新时间；支持按债券类型分组折叠展示；支持拖拽调整关注顺序并在刷新页面后保持顺序（持久化到后端或 localStorage 需前后端协商实现）。列表字段映射需修复 `type`、`rating` 空白问题，并确保与聚合行情页品种标签样式一致。

**F-07 [Feature迭代]**  
国债期货页 `/futures` 的基差分析目前仅为表格。请在「基差分析」区域表格上方增加基差时序折线图：默认展示 T 主力合约近若干时间点基差走势（可用 seed 数据聚合或后端新增轻量接口）；支持通过下拉切换 TF、TS 合约；折线图与下方表格选中行联动高亮。Y 轴单位清晰标注，基差穿越零轴处加参考线。

**F-08 [Feature迭代]**  
收益互换页 `/swaps` 存在前后端字段不一致导致「期限」「最小名义本金」等列空白。请全面修复字段映射（`tenor`、`notional_min` 等），并为「参考债券」列增加可点击跳转到债券详情的链接。新增按债券类型、互换方向（支付/收取固定）筛选；同一债券下若有多条报价，最优 swap rate 行背景高亮。顶栏增加当前列表条数统计。

**F-09 [Feature迭代]**  
系统管理中的行情源管理页 `/admin/sources` 开关可能无法正确持久化，且类型、启用状态列显示异常。请修复与后端 `source_type`、`is_enabled` 的字段对齐，PUT 请求体使用 `{ is_enabled: boolean }`。在列表中增加「最后更新时间」「模拟延迟(ms)」列（延迟可由后端基于 seed 或随机生成）；支持勾选多个行情源后批量启用/禁用；操作成功后表格即时刷新且给出 Ant Design 消息提示。

**F-10 [Feature迭代]**  
用户管理页 `/admin/users` 目前为只读列表且状态列显示错误。请修复 `is_active` 与前端 `status` 的映射，展示为「启用 / 停用」标签。增加按角色（admin / trader / viewer）筛选与用户名关键字搜索；admin 可对用户执行启用/停用切换（需新增后端 PATCH 接口）；表格展示「最后登录时间」列（若库中暂无该字段，可扩展 User 模型并在登录时更新）。非 admin 用户访问该路由应被前端路由守卫拦截并 redirect 到看板。

**F-11 [Feature迭代]**  
债券详情页「各源报价对比」Tab 需要更强的对比能力。请增加按行情源类型（xBond / 经纪商 / 交易所等）多选筛选；增加「仅看最优价 / 全部报价」切换；在表格右侧或下方增加横向 Spread 条形图，直观对比各源买价或收益率。报价时间显示为相对时间（如「3 分钟前」）并在 hover 时展示绝对时间。切换筛选条件时保持 Tab 状态不丢失。

**F-12 [Feature迭代]**  
交易员需要在任意页面快速定位债券。请在 `AppLayout` 顶栏增加全局搜索框：输入债券代码或名称时 debounce 调用 `/api/bonds` 搜索，下拉展示最多 8 条联想结果（代码、名称、品种）；键盘上下键选择、Enter 跳转详情，Esc 关闭。搜索历史（最近 5 条）存入 localStorage 并在空输入时展示。移动端宽度下搜索框可收折为图标点击展开。

**F-13 [Feature迭代]**  
看板「行情异动提醒」列表目前信息密度不足。请为每条提醒增加 severity 等级（高/中/低）并用左边框或标签颜色区分；支持单条「标记已读」与「全部已读」；已读项样式变淡且默认折叠到「已读」分组。点击提醒条目跳转对应债券详情。若后端 `/api/dashboard/alerts` 尚未包含 severity，请扩展接口并在 seed 中补充示例数据。

**F-14 [Feature迭代]**  
聚合行情页筛选能力需对齐专业终端。请在现有关键词、品种、评级基础上，增加剩余期限区间（如 1-3Y、3-5Y、5-10Y、10Y+）、票息率范围滑块、评级多选（AAA、AA+ 等）。筛选条件变化时更新 URL query 参数，复制链接可在新标签页还原相同筛选状态。重置按钮一键清空全部条件。表格分页与筛选组合时页码应正确重置为第 1 页。

**F-15 [Feature迭代]**  
成交记录页需要导出与明细查看能力。请在 `/trades` 增加「导出 CSV」按钮，按当前筛选条件（行情源类型、债券类型、分页前的全量或当前页需明确行为）下载 CSV 文件，表头为中文。点击表格某行弹出右侧 Drawer 展示成交详情：债券名称、代码、方向、价格、收益率、量、金额、对手方、行情源、成交时间。Drawer 内提供「查看债券详情」快捷按钮。

**F-16 [Feature迭代]**  
债券详情页「成交记录」Tab 请增强为可分析视图：在表格上方增加按日成交量柱状图（近 7 日），柱颜色区分买入/卖出方向；表格增加方向图标列；支持按方向、金额下限筛选。图表与表格共用同一 bond_id 数据源。无成交时图表区展示空态插画而非空白。

**F-17 [Feature迭代]**  
看板顶部六张统计卡片目前为静态数字。请为每张卡片增加较昨日变动趋势（上升/下降箭头及百分比或 BP），点击卡片跳转到对应业务页（如「今日成交笔数」→ `/trades`，「活跃债券数」→ `/market`）。卡片加载时使用 skeleton 占位。若后端 overview 接口缺少同比字段，请扩展 `/api/dashboard/overview` 并在 seed 逻辑中提供可对比的基准值。

**F-18 [Feature迭代]**  
登录页 `/login` 体验需要 polish：增加「记住用户名」勾选（仅保存用户名至 localStorage，不保存密码）；登录失败时在表单项下方展示后端返回的具体错误信息；登录成功后顶栏用户区展示「欢迎回来，{username}」及角色中文标签（管理员/交易员/只读）。登出时增加二次确认 Modal，确认后清除 token 并跳转登录页。可选：增加简单登录表单 front-end 校验（空用户名/password 禁止提交）。

**F-19 [Feature迭代]**  
聚合行情列表需让交易员一眼识别已关注债券。请集成 favorites 状态：进入 `/market` 时批量查询当前用户收藏列表，在已收藏行首列展示实心星标，未收藏展示空心星标；点击星标可就地收藏/取消收藏（调用现有 favorites API），操作后星标即时切换且「我的关注」页同步。viewer 角色若不允许写操作，点击星标应提示权限不足而非静默失败。

**F-20 [Feature迭代]**  
债券详情页「详细报价」Tab 报价条目较多时难以定位。请为该 Tab 增加报价源 logos/缩写标识列、报价有效状态（有效/过期，过期行灰显）、按报价时间排序切换；支持展开行查看 counterparty、quote_id 等次要字段。顶栏增加「仅 xBond」「仅经纪商」快捷 Chip 筛选。过期判定规则可在后端根据 quote_time 与当前时间差 > 30 分钟标记，或前端按统一规则计算，需前后端一致。

## 0-1 模块构建（20 题）

**B-01 [0-1]**  
价格预警模块：交易员可针对单只债券设置收益率或净价的上下限条件，规则在独立管理页集中维护，也可在债券详情页快速设警。系统持续比对最新聚合行情，条件满足时生成触发记录；顶栏铃铛展示未读触发数量，下拉可浏览最近告警并跳转至对应债券。规则支持启用/停用、编辑与删除，触发历史可按债券与时间筛选回溯。

**B-02 [0-1]**  
债券对比模块：独立对比页允许通过搜索添加最多四只债券，以并排表格对照展示代码、名称、品种、评级、剩余期限、最优买卖价、最新成交价及近七日成交量等核心指标。最优价、收益率等关键字段自动标出优劣差异；支持移除单券、一键清空，对比清单可通过链接参数分享给同事，打开链接即可还原同一组对比。

**B-03 [0-1]**  
自定义 Watchlist 分组模块：在现有「我的关注」之外，提供可自由命名的债券分组能力，例如持仓关注、新发债跟踪等。每组可独立增删债券、调整成员顺序；侧边栏展示分组快捷入口，进入后只浏览该组债券及其最新最优价。分组本身支持重命名与删除，组内列表展示与聚合行情页风格保持一致。

**B-04 [0-1]**  
WebSocket 实时行情模块：聚合行情列表与债券详情页的报价数据改为服务端主动推送更新，替代纯手动刷新。价格上行与下行时对应单元格短暂闪烁绿/红提示；连接异常时自动重连，顶栏以状态点标示当前连接情况（正常、重连中、断开）。推送范围随页面可见债券动态订阅，离开页面后释放订阅以减轻负载。

**B-05 [0-1]**  
Redis 缓存模块：看板概览、最优报价排行、债券聚合行情等高频只读接口接入缓存层，缓存有效期可通过环境配置调整。接口返回数据中附带数据更新时间，前端在看板与详情页以小字展示「更新于 xx:xx:xx」。管理员可在后台一键刷新指定范围的缓存，便于行情源变更或数据修正后即时生效。

**B-06 [0-1]**  
操作审计日志模块：系统自动记录用户登录、收藏变更、行情源启停等关键操作，每条日志包含操作人、动作类型、操作对象、详情摘要、来源 IP 与发生时间。管理员在专属页面按时间倒序浏览，可按用户、动作类型、日期区间筛选，详情区可展开查看完整操作内容。日志只读不可篡改，分页加载以支撑长期积累。

**B-07 [0-1]**  
债券计算器模块：独立工具页提供债券估值辅助计算，输入面值、票息率、付息频率、结算日、到期日后，用户可指定净价求到期收益率，或指定收益率反推净价。结果区同步展示应计利息、全价、麦考利久期与修正久期等指标；表单字段实时校验，非法日期或负数输入给出明确提示。侧边栏「工具」菜单可直达该页。

**B-08 [0-1]**  
利差分析模块：用户选定基准债与目标债（或最多五只组合），页面绘制利差随时间变化的折线图，并给出当前利差、历史均值、标准差及简版 Z 分数参考。支持在收益率利差与净价利差两种口径间切换，下方明细表列出各日时点具体数值。选债过程支持关键字搜索联想，图例区分各组合曲线。

**B-09 [0-1]**  
可配置工作台模块：看板页的统计卡片、收益率曲线、热门债券、异动提醒四个区块支持个性化编排——拖拽调整顺序、隐藏不需要的区块。布局偏好保存至用户账户，下次登录自动恢复；提供「恢复默认布局」一键重置。编辑模式下各区块显示拖拽手柄与显隐开关，退出编辑后恢复正常浏览体验。

**B-10 [0-1]**  
消息中心模块：顶栏铃铛汇总系统公告、行情异动、价格预警触发、管理员广播等多类消息，角标显示未读数量。消息收件箱按类型分栏浏览，支持单条标记已读与全部已读；点击消息跳转至关联页面（债券详情、预警列表等）。各业务模块产生通知时走统一写入通道，保证消息格式与已读状态一致。

**B-11 [0-1]**  
债券日历模块：月视图日历标注付息日、到期日、新发债等事件，有事件的日期以圆点或底色区分。点击某日弹出当日事件列表，可按债券品种筛选；事件条目可跳转至对应债券详情。事件数据与现有债券库关联，日历支持前后月份切换，当月事件密集日期一目了然。

**B-12 [0-1]**  
研报摘要模块：研报列表展示标题、发布机构、日期、评级观点摘要及关联债券；支持按债券代码或名称搜索过滤。点击进入详情页阅读完整摘要正文，文末以标签形式列出相关债券并可跳转详情。侧边栏增加研报入口，演示数据覆盖多只现有债券的研究观点 mock 内容。

**B-13 [0-1]**  
经纪商报价板模块：以紧凑网格呈现主要经纪商对热门债券的最新 bid/ask 双边报价，行对应债券、列对应经纪商，单元格内买卖价分行展示并标注报价距今时间。整体视觉区别于普通宽表，适合盯盘场景；页面定时自动刷新，也可接入实时推送增量更新。点击任意报价格跳转至该债券详情页。

**B-14 [0-1]**  
历史行情回放模块：用户选定债券后，通过时间轴滑块在多个历史快照间切换，查看各时刻各行情源的报价与成交情况。滑块旁显示当前快照时间点，主体区域渲染该时刻的多源报价对比表；可选开启相邻快照差异对比，高亮 bid/ask 变动幅度。热门债券预置若干历史快照供回放浏览。

**B-15 [0-1]**  
快捷键模块：全局注册常用键盘快捷操作，例如 Ctrl+K 唤起搜索、连续按 G 再按 D 跳转看板、G+M 跳转聚合行情等，至少覆盖八组常用组合。按问号键或点击顶栏帮助图标弹出快捷键速查面板；另设静态说明页集中列出全部快捷键及含义。输入框聚焦时暂停页面跳转类快捷键，避免与文字输入冲突。

**B-16 [0-1]**  
主题切换模块：提供深色、浅色、高对比度三套界面主题，顶栏下拉即可切换，偏好同时保存至本地与账户设置。浅色与高对比度模式下侧边栏、表格、图表配色协调可读；切换过程无整页闪烁，当前浏览位置保持不变。外观设置页可预览三套主题的代表色板并选择默认主题。

**B-17 [0-1]**
